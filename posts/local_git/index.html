Getting started with the version control

May 22, 2017 4-minute read
git • version-control

Using a local folder and git.

 
	# create a directory -- eg: recipe
	mkdir recipe
	
	# change the directory i.e. cd to the recipe
	cd recipe
	
	# initiate git on the current directory 
	git init
	
	# use your username and email account information, the one that you have with git-hub/gitlab
	git config --global user.name "-----"
	git config --global user.email "-----"
	git config --global color.ui "auto"
	
	# check the current status
	git status
	
	# create a file: eg. pizza.txt
	nano pizza.txt
	
	# once the screen open, write something or code...
		Recipe for making pizza
		
	# save and exit from nano
		ctrl O >> enter >> ctrl X
		
	# check the current status
		git status
		
	# add the file to git
		git add pizza.txt

	# check the current status
		git status
		
	# commit on the file
		git commit –m "started list for pizza. "
	
	# check the current status
		git status
		
	# check the logs for files
		git log
		
	# now open your pizza file with nano text editor and make some changes on pizza receipe
	nano pizza.txt
		eg. List of ingredients for pizza
			Crust
			Toamato paste
			Cheese
	ctrl O >> enter >> ctrl X
	
	# check the current status
		git status
		
	# add the file to git
		git add pizza.txt
		
	# check the current status
		git status
	
	# commit on the file
		git commit –m " added ingredients for pizza"
		
	# check the logs for files
		git log
	
	# again make some changes on the pizza recipe
		nano pizza.txt
			make some changes
		ctrl O >> enter >> ctrl X
		
	# check the current status
		git status
	
	# shows the different version or tracked
		git diff pizza.txt 
		
	# compare the most recent with the second most recent.
		git diff HEAD pizza.txt 
		
	# compare the most recent to two version back
		git diff HEAD~1 pizza.txt
		
	# here d11b1 is the first 5 letters/numbers of commit
		git diff d11b1~1 pizza.txt 
		
	# now remove the pizza.txt -- accidental case where you delete your file/code. DON;T PANIC!!! git has solution
		rm pizza.txt
		
	# retrieve the lost file ....cool...!!!
		git checkout HEAD pizza.txt
		
	# check the content 
		cat pizza.txt

Now you have mastered the version control on the local machine. Let’s learn how to link the local folder to git-hub repository.

	# make an account in git-hub webpage
		create a new repository ~ recipe
		copy url

	# In terminal
		git remote add origin <url>
		git remote –v #Verbose gives more information for the step
	
	#  # set url as the main branch
		git push –u origin master
		
	# go to git-hub webpage and look your recipe folder/repository.
		
	# In terminal 
		nano pizza.txt
		make changes and ctrl O >> enter >> ctrl X
		
	# Follow the steps below
		git status pizza.txt
		git add pizza.txt
		git status pizza.txt
		git commit –m "added steps to make pizza"
		git status pizza.txt
		
	# applies to whole folder, and update changes to the web repository as well.
		git push 
		
	# Follow following steps to clone from your git-hub page
	# in terminal
	cd ..
	
	# remove the whole folder 
	rm –rf recipe
	
	# notice the folder has been removed / deleted
	ls
	
	# clone using remote git-hub repository 
	# retrieve your deleted folder form web repository.
	git clone <url>

Now lets see how to share folder/ script with collaborators.

	# In web git-hub 
	# Add new collaborators [search by usename or email]

	# Collaborator has to clone the shared repository in his/her computer.
		Copy url [clone and download tab in github webpage]

	# Collaborator has to do this step in his/her terminal
		Go to the folder where you want to clone 
		git clone <url>
	
	# Now its your collaborators turn to work on code.
		nano pizza.txt
		make changes and ctrl O >> enter >> ctrl X
		git add pizza.txt
		git commit –m "make some changes on the direction"
		git push

	# Now lets pull the changes made by your collaborators to your computer 
		git pull
		git status pizza.txt

	#If you don’t pull the changes made by your collaborator’s and make changes. Then you will see following warnings.
	nano pizza.txt
		make changes and ctrl O >> enter >> ctrl X
	git add pizza.txt
	git commit –m "added more direction"
	git push
	"will give you message that you have not pulled the changes"
