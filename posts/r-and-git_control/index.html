Migrate your R code to remote git server

April 10, 2017 2-minute read
version control • git • R

Using git-hub. Similar code/steps can be used to link the local folder with git-lab.

-	Go to Rstudio > Global Option > Git/SVN > Create RSA Keys > View public keys

-	Copy RSA keys

-	Go to git-hub webpage > settings > SSH and GPG keys > Add keys

-	Rstudio > New Project > New Directory > Empty Project

-	Directory name: e.g. githubcapstone

-	Create a git-repository

-	Create project ## Note: Next to history you should see Git tab

-	In Rstudio: add a folder called data. 

Usually data used for analysis can be stored here. Lot of people also keep data under 
.gitignore folder

-	Rstudio> tools > Shell

Initialize your folder to git and your gitlab information

	# Run the following code in RStudio-shell
	git init
	git config --global user.name "--------"
	git config --global user.email "-------"
	git config --global color.ui "auto"
	git status
	
Now add a new repository e.g. githubcapstone in github web then copy http for the new repository.
 
-	Rstudio> tools > Shell


Run the following code in RStudio-shell

	# Link git hub account with your local folder in R.
	git remote add origin <https://…..git> # https:// --- copied from git-hub
	git remote –v
	git commit –m "first upload to gitlab."
	git push -u origin master
	
	prompted with username: "------"
	prompted with password: "------"
